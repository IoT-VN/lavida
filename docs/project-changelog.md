## Change Format

### Change Types
- **Added** for new features
- **Changed** for changes in existing functionality
- **Deprecated** for soon-to-be removed features
- **Removed** for now removed features
- **Fixed** for any bug fixes
- **Security** in case of vulnerabilities

### Contributing
When making changes, please follow this format:
1. Create a new section for the version/release
2. Use the appropriate change type
3. Include description and any relevant details
4. Link to related issues if applicable

### Versioning
We use [Semantic Versioning](https://semver.org/) for versioning:
- **Major (X.0.0)**: Incompatible changes
- **Minor (X.Y.0)**: New features, backward compatible
- **Patch (X.Y.Z)**: Bug fixes, backward compatible
